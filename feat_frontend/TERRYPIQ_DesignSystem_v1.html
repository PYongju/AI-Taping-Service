1. Introduction

1.1 Overview

TERRYPIQ 디자인 시스템은 무릎 테이핑 추천 서비스를 일관되고 신뢰감 있게 제공하기 위한 기준입니다. 모든 화면과 컴포넌트는 사용자의 빠른 행동을 돕고, 안심하고 사용할 수 있는 경험을 만드는 것을 목표로 합니다.

4. Patterns

💬
4.1 Conversational Flow
입력→선택→로딩→결과의 단방향 대화 플로우를 사용해요.
📦
4.2 Empty State
데이터가 없을 때 Terry 일러스트 + 행동 유도 CTA를 표시해요.
⚠️
4.3 Error State
문제 상황에서 "문제 → 해결 방법" 구조의 안내 문구를 사용해요.
🔒
4.4 Permission / Privacy
개인 정보 수집 전 목적을 명확히 안내하고, 건너뛰기를 항상 제공해요.

5. Terry (Character System)

Terry는 TERRYPIQ의 브랜드 마스코트입니다. 테이프 롤을 모자처럼 쓴 작은 쥐 캐릭터로, 오렌지 단일 배색을 사용해요. Terry는 말하지 않습니다 — 감성과 브랜드 친밀감은 비주얼과 포즈로만 전달하고, 시스템 문구는 UX Writing Guide 보이스톤을 따릅니다.

6.1 Voice & Tone 원칙

6.2 Do / Don't

✅ DO
• 해요체를 사용해요
• 능동형 문장을 써요
• 행동 → 이유 순서를 지켜요
• 한 줄 = 하나의 행동
• 1줄 = 1행동형 원칙
❌ DON'T
• 의료 진단 표현 금지
• 확정 표현 금지 ("가장 좋은 방법입니다")
• 책임 회피 표현 금지
• 과다감정표현 금지 ("괜찮아요")
• 수동형 금지 (~됩니다, ~되었어요)

6.3 CTA 규칙

규칙 ❌ 명사형 ✅ 동사형
동사형 강제 다음 / 완료 / 확인 다음으로 넘어갈게요 / 완료했어요 / 확인할게요
결과 명확화 확인 / 다음 이 방법으로 시작할게요 / 사진 올리러 갈게요
닫기 vs 취소 취소 닫기 (바텀시트·다이얼로그 왼쪽 버튼)
